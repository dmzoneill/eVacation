The following pages are accessible by clicking on the Admin Menu tab.

Select User
Available to: Anyone who is set up as e-Vacation Administrator.
NOTE: An e-Vacation administrator cannot edit his/her own details.

Description: By selecting a single employee, e-Vacation allows the Admin User to perform Administration functions (e.g. entering user information, carry-over, leave, activate ELP, and set access rights for an employee).

To select a user:
1.  Enter a WWID or click on the Search Icon to search for WWID.
2.  Click on the Select button.

e-Vacation validates the entry and, if valid, displays the User Profile page of the employee. All the administrative functions also become available to the administrator.
User Profile
Available to: Anyone who is set up as e-Vacation Administrator.

Description: Allows the Admin User to create or change the e-Vacation User Profile details for the selected employee.

To set User Profile for the selected employee:
1.  Enter the employee End of Contract Date if it is known - either enter the end of contract date in the specified format (e.g. 1 Jan 2006) OR click on the calendar icon to select a date.
2.  Tick the Leave Tracking Enabled box (if not already ticked) to set the employee up as an e-Vacation user.
3.  Tick the Administrator Access box - ONLY if the employee is to have administration rights to e-Vacation.
4.  Click on the Save Changes button to save the entry.

If entries are invalid, a red message will be displayed (click the here link to find out the reason/s for invalid entry/entries).

NOTE: Access to e-Vacation is not automatically granted. New Intel Shannon blue-badge employees will be prompted to request an administrator to set up access when he/she tries to access e-Vacation. The new employee will gain access only when an administrator saves the employees user profile i.e. clicks on the Save Changes button.
Carry Over / Exceptions
Available to: Anyone who is set up as e-Vacation Administrator.

Description: Allows the Admin User to enter
a) the number of vacation days to carry over to a specified year for the selected employee.
b) any exceptions.

To enter Carry Over / Exceptions:

1.  Select the type of carry-over from the pull-down menu. There are two choices:
a)  Carry Over EOY (End Of Year):
This occurs annually, and is given where an employee has not used all his vacation the current year.
b)  Exceptions:
This is used to enter to make adjustments to an employee’s annual entitlement where they have an exceptional allowance (e.g. seniority is based upon a non-standard set of rules)
2.  Enter a Year – i.e. the Year for which this carry over applies to.
3.  Enter the number of days to apply.
4.  (optional) Enter any comments (NOTE: Release 1 of e-Vacation does not allow the employees to view these comments).
5.  Click the Update Carry Over button to save.

Repeat Steps 1 to 5 to edit an entry.
NOTE: you are able to enter negative exceptions if you need to make deductions from a leave entitlement (for example if someone has less annual allowance than the standard entitlement).

If entries are invalid, a red message will be displayed (click the here link to find out the reason/s for invalid entry/entries).

Once a Carry Over / Exception is applied to a specified year, it automatically adds the value to the employee's entitlement for that year. For example, a Carry Over EOY of 5 days was applied to Year 2002 on 22 Dec 2001 for Adam Smith by an e-Vacation Administrator. When Adam Smith goes into e-Vacation, his My Leave Summary, for 2002 will show 5 days added to Carry Over (from previous year).

IMPORTANT NOTE: Release 1 of e-Vacation does not check and make adjustments for carry-over (EOY) entitlement which has not been taken by 31st of March in the year it applies. It is the responsibility of e-Vacation Administrators to manually adjust the values to account for any days not taken. To do so, it is recommended that the adjustments are made to the Carry-over EOY field.
For example; Joe Bloggs had 5 days carried over into 2002, but took only 3 days vacation by 31st March. To account for the 2 days lost, change the Carry-Over EOY figure to 3 days from 5 days (i.e. Joe looses 2 days entitlement).
Leave
Available to: Anyone who is set up as e-Vacation Administrator.

Description: Allows the Admin User to add a Leave Request on behalf of the selected employee. It also displays the employees Leave Summary and Leave Requests for a selected year.

Unlike Leave Requests raised by employees themselves, requests raised by an Admin User will not require approval. It is taken that the leave will have already been approved / acknowledged by the employee's manager, in accordance with local HR processes.
NOTE: Please ensure that you have evidence that the manager has approved the request before it is entered.

To Submit a Leave Request:
1.  Select a Leave Type from the pull-down menu.
2.  Enter a Start Date AM/PM – click on the calendar icon to enter the date and select the time (morning/afternoon) from which you will start your leave.
3.  Enter an End Date AM/PM – click on the calendar icon to enter the date and select the time (morning/afternoon) that your leave will end.
4.  (optional) Enter Comments.
5.  Click on the Submit Leave Request button.

On submitting the request e-Vacation validates the entries. If entries are invalid, a Red Message will be displayed (click the here link to find out the reason/s for invalid entry/entries).

To Delete a Leave Request:
1. Scroll down to the Leave Request section.
2. Click on the View link next to the leave request.
3. Click on the Delete button in the View Leave Request section.
4. Click on OK.

Please be aware that e-Vacation does not allow tracking of deleted requests.

How does e-Vacation calculate my leave?

The number of working days within a leave request is automatically calculated.
For example:
    Start Date is 27 Apr 2001 (Friday) AM
    End Date to 30 Apr 2001 (Monday) AM

    Working days = 1.5 Days

e-Vacation knows to exclude Saturdays, Sundays and Public Holidays (see the Public Holidays page for details)

Click on the Rules Guide for further information about the rules build into e-Vacation.
ELP
Available to: Anyone who is set up as e-Vacation Administrator.

Description: Allows the Admin User to activate and change details of an Extended Leave Program (ELP) for a selected employee. e-Vacation displays the appropriate ELP section (ELP Activate, Active, Matured) depending on the employees ELP status. The activation is submitted to the confirmation sent by ESC that the ELP request from the employee has been approved by his/her manager.

Activate ELP section: This is used to activate a new Extended Leave Program.

To Activate an ELP:
1.  Enter the Activation Date – a day during the first year that ELP is banked. When entering historical ELP information (e.g. the employee first banked ELP in 1999) we suggest entering 1st Jan of the year in which ELP was first banked (e.g. 1st Jan 1999).
2.  Enter the number of days (usually 5 or 10) to be banked in year 1.
3.  Enter the number of days that will be banked in total across the ELP period (this is normally 25 days).
4.  Click the Activate ELP button to activate this ELP.

Once activated, e-Vacation validates the entries. If entries are invalid, a red message will be displayed (click the here link to find out the reason/s for invalid entry/entries).

NOTE:  e-Vacation automatically deducts 5 days in every subsequent year until the total days (normally 25 days) is reached. The total number of days to be banked minus days banked on activation MUST BE DIVISIBLE BY 5.

If 10 or more days are entered for Year 1, the employee is given 3 years to take the ELP once it is matured. If less than 10 days are entered, the expiry period is reduced to 2 years.

Once an ELP is activated, e-Vacation automatically deducts the number of days banked for each year from the employees annual leave balance.

Active / Matured ELP sections: Displays Active and/or Matured ELP details including days banked/year, activation date, maturity and expiry dates. Once the required number of days has been banked, ELP matures and becomes available to take. Once today's date becomes greater than the matured ELP's expiry date, the matured ELP record will no longer be shown.

To edit an ELP:
1.  Click on the Edit ELP Activation Details button.
2.  Edit the activation date / days banked on activation / total to be banked, as required.
3.  Click on the Save ELP button.

Active ELP Reliefs section: Used to provide relief from banking 5 days in any year during the ELP period.

To enter an ELP Relief:
1.  Select the Year that this ELP relief is to be applied to (select from the list as it displays all valid years for this ELP).
2.  Click the Add Relief button to apply this relief to the specified year. E-Vacation will save this relief, changing the number of days banked for the selected year to 0, and extending the ELP period by 1 year.

To Remove an existing ELP Relief
1.  Select the Year that ELP relief has been applied (the pull-down list all years where ELP has been applied).
2.  Click the Remove Relief button. e-Vacation removes this relief.

ELP Leave Request Form: This will be displayed if you have a matured ELP. It is used to book ELP vacation. The employee can also book ELP vacation using e-Vacation.

To book an ELP vacation:
1.  From the ELP Request Form, enter the start date for the vacation.
NOTE: you can only see the ELP request form on screen if the ELP is matured (and not expired).
2.  (optional) enter any comments.
3. Click the Submit Leave Request button.

e-Vacation will automatically work out the end date of the vacation based upon the number of days in the employees ELP bank.

To delete an ELP vacation:
e-Vacation does not include an obvious facility to delete ELP (i.e. a Delete ELP button). However, it is still possible to delete an ELP if one is inappropriately entered. To do so:
1.  Click on the Edit ELP Activation button.
2.  Change the Activation Date to 1st Jan 1990.
3.  Click on the Save ELP button.

Because the activation year is set to 1990, the ELP will expire and not be displayed.
Reports
HR/Payroll Report

Available to: Anyone who is set up as e-Vacation Administrator.

Description: Allows the Admin User to run HR/Payroll Report. The report will show you a list of employees with Leave Period/s (regardless of the Status) within the Report Requested Dates (including those Leave Period which overlaps the Report Requested Start Date and End Date).

To run a report:
1.  Enter a start date (use the calendar or type it in the format DD MMM YYYY).
2.  Enter an end date (use the calendar or type it in the format DD MMM YYYY).
3.  Click on the Generate Report button.

A new window will appear in which the report will be displayed.

NOTE: The report may take several minutes to run. You can minimise the window to run in the background.

To print the report:
1.  Click on the Window's Main Menu - File -> Print -> OK.

To save the report to Excel:
1.  Click on the Window's Main Menu - File -> Save_As menu option.
2.  Select a file directory.
3.  Enter a file name.
4.  Select the .xls extension.
5.  Save the file.

Note: HR/Payroll Report will select data starting on 1 Jan going back 2 fiscal years from the Report Start Date e.g. Report Start Date is 6 June 2001, it will look at leave period/s starting on 1 January 2000. From the data returned the report will check each leave period to see if it is within the Report Requested Dates for display.
Database Query
Send query to the MS-SQL database

Available to: Anyone who is set up as e-Vacation Administrator.

Description: Allows the Admin User any type of SQL request on the MS-SQL database. (including SELECT, INSERT, UPDATE, CREATE,...)

To send a query:
1.  Write a SQL query in the text box. (use the examples link to see how to write a query)
2.  Click on the execute button.

A new window will appear in which the query and the result will be displayed.

NOTES:
- Some characters are not supported in queries (e.g. '+'), you can check that the query executed is the same as the one asked.
- You can put several queries in the text box, all will be executed but only the result of the first one will be displayed.
